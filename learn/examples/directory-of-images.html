Loading a Directory of Images

This is something that is often useful when writing computer vision applications, because there are times when a camera isn't readily available, but you do have a supply of image files to work with. This example will show you how to easily load a directory of images so you can work with them.

When working with images, it's also sometimes useful to work with a set of images. An image set of similiar items can help you define the common features or characteristics between one item and the next. You can then take these feature definitions, and build a computer vision application that sorts through images and determines if the submitted image meets the feature criteria or not.

Note: There is a list of free, online data sets in the SimpleCV GitHub repository: http://github.com/ingenuitas/SimpleCV/wiki/List-of-know-open-data-sets-for-testing

For this example, you will need a set of images located in a directory on your computer. It can be the same directory as the script, or a different one. As written, the example code expects the images to be in the '/tmp/cats/' directory, and to have the .png file extension. You can simply change these values to meet your own situation.

Here is the code to load a directory of images, and then display them:

import os
import glob
import time
from SimpleCV import Image

# Settings

# Change this to the correct directory path for your images - or 
# remove the '/tmp/cats' value if the images are in the same directory 
# as the script
my_images_path = "/tmp/cats/" 

# Change this to the correct file extension for your images
extension = "*.png"


# Main Program

if not my_images_path:
        path = os.getcwd() 
else:
        path = my_images_path

directory = os.path.join(path, extension)
files = glob.glob(directory)

for file in files:
        new_img = Image(file)
        new_img.show()

	# Wait for 3 seconds before looping to the next image file
        time.sleep(3)

import os
import glob
import time

The three lines import some standard Python libraries. The os library is used to work with the operating system to find the current working directory. It's also used to create a pattern to match against the image path names. The glob library, on the other hand, is used to capture all of matched image file paths. The time library is used to create a small timer that delays the display of each image by 3 seconds.

print os.__doc__

After the line loading the SimpleCV Image library, there are a few lines that you can use to configure the script to meet your needs:

# Settings

# Change this to the correct directory path for your images - or 
# remove the '/tmp/cats' value if the images are in the same directory 
# as the script
my_images_path = "/tmp/cats/" 

# Change this to the correct file extension for your images
extension = "*.png"

The lines starting with the hash symbol (#) are simply comments in the script, and are ignored by the Python interpreter. The other two lines create variables that are used later in the script to find the appropriate image files.

The next block of code is an if...else statement which checks to see if a value has been set for the my_images_path variable. If my_images_path is empty, the application uses the getcwd() function from the os library to find what the path is for the directory the script is located in. 

if not my_images_path:
        path = os.getcwd() 
else:
        path = my_images_path

Once the application has the correct path, the code builds out a pattern for the image paths by appending the file extension to the path. This pattern is then used to find all the files that match it. In our example, this pattern would be '/tmp/cats/*.png'.

directory = os.path.join(path, extension)

The glob() function is then used to create a list of all of the file names that match the pattern:

files = glob.glob(directory)

The next block of code then loops through all of the files:

for file in files:
        new_img = Image(file)
        new_img.show()

        # Wait for 3 seconds before looping to the next image file
        time.sleep(3)

The Image() function is used in the first line of the while block to load in each image file:

        new_img = Image(file)

The show() function is used to display the image to the user. Finally, the last line just uses the sleep() function from the time library to pause the while loop for 3 seconds before loading up the next image file.

If you want to play with this code yourself, the file is named directory_of_images.py in the example zip file.
